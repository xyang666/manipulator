## 论文声称 vs 代码实现对比

### 1. 零空间策略学习（核心贡献1）

**论文声称：**
- 策略仅输出零空间内的自运动速度 $\dot{q}_0$
- 通过零空间投影保证不影响末端跟踪
- 降低探索空间维度（从7维到1维）

**代码实现：**
```python
# kinematics.py:161
N = np.eye(self.n) - Jpinv @ J  # 零空间投影矩阵
dq_null = N @ dq0               # 投影到零空间
dq_combined = dq_task + dq_null # 组合速度
```

**评估：**
- ✅ **零空间投影已实现**：`N = I - J†J` 是标准的零空间投影矩阵
- ⚠️ **维度理解偏差**：
  - 论文说"1维零空间"是指零空间的**秩为1**（7-6=1）
  - 但策略输出仍是7维向量 `dq0`，然后通过 `N @ dq0` 投影到1维零空间
  - 这是**正确的实现方式**，因为零空间投影需要在7维空间中操作
- ✅ **理论优势成立**：虽然action是7维，但 `N @ dq0` 保证了只在零空间内运动，不影响末端

### 2. 物理信息增强（核心贡献2）

**论文声称：**
- 将拉格朗日动力学约束嵌入奖励函数
- 动力学一致性损失：$r_{\text{dyn}} = -\|\tau_{\text{pred}} - \tau_{\text{actual}}\|^2$
- 提升样本效率53%，力矩平滑度60%

**代码实现：**
```python
# physics_policy.py:167
tau = self.dynamics.compute_torque(q, dq, ddq)  # τ = M(q)ddq + C(q,dq)dq + g(q)
violation = F.relu(tau.abs() - tau_max)
loss = (violation ** 2).mean()
```

**评估：**
- ⚠️ **实现与论文不一致**：
  - 论文：计算 $\|\tau_{\text{pred}} - \tau_{\text{actual}}\|^2$（预测力矩与实际力矩的差异）
  - 代码：计算 $\|\text{relu}(|\tau| - \tau_{\text{max}})\|^2$（力矩超限惩罚）
- ⚠️ **理论依据不同**：
  - 论文声称：通过动力学一致性引导策略学习物理可行轨迹
  - 代码实际：通过力矩限制约束防止超出硬件能力
- ✅ **效果可能存在**：虽然实现不同，但力矩限制约束确实能提升训练稳定性
- ❓ **因果关系待验证**：53%样本效率提升是否来自这个损失，需要消融实验验证

## 关键问题总结

### 问题1：物理损失定义不一致（中等严重）
- **论文**：动力学一致性损失 $\|\tau_{\text{pred}} - \tau_{\text{actual}}\|^2$
- **代码**：力矩超限惩罚 $\|\text{relu}(|\tau| - \tau_{\text{max}})\|^2$
- **影响**：理论动机与实际实现不匹配，审稿人可能质疑
- **修正方案**：
  - 选项A：修改论文，诚实描述为"力矩限制约束"而非"动力学一致性"
  - 选项B：修改代码，实现真正的动力学一致性损失（需要获取MuJoCo的实际施加力矩）

### 问题2：碰撞检测范围（轻微）
- **论文**：声称使用SDF计算所有连杆距离
- **代码**：主要检测末端位置（`d_obs = self.sdf.min_distance(x_ee, ...)`）
- **影响**：避障效果可能被高估
- **修正方案**：在论文中说明"以末端为主要检测点"或扩展代码检测所有连杆

### 问题3：实验对比缺失（严重）
- **论文**：声称对比5个方法（RRT*、CHOMP、PPO、SAC-Baseline等）
- **代码**：只有SAC实现
- **影响**：无法验证"提升12-18个百分点"的核心结论
- **修正方案**：
  - 必须实现至少2个baseline（SAC-Baseline、PPO-Baseline）
  - 或降低论文声称，只对比"有/无物理约束"的消融实验

### 问题4：动态障碍未实现（中等严重）
- **论文**：声称在动态障碍场景下测试，成功率81.3%
- **代码**：障碍物在reset时固定，不会运动
- **影响**：实验结果不可信
- **修正方案**：
  - 实现动态障碍（在step中更新障碍物位置）
  - 或删除论文中关于动态障碍的所有声称
