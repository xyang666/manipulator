## 1. 项目状态

- 代码框架完整（环境、智能体、奖励、轨迹生成、训练循环）
- **checkpoints/ 为空** — 尚未进行任何训练
- 已有 100 个训练场景 (`results/trajectories_obs.json`，ID 0-99)，每场景 5 个障碍物
- 支持 SAC（默认，off-policy）和 PPO（on-policy）

### 核心架构

| 组件 | 维度 | 说明 |
|------|------|------|
| 观测 | 28D | `[q(7), dq(7), x_ee(3), x_d(3), dx_d(3), d_obs(1), w(1), obs_dir(3)]` |
| 动作 | 7D | `[Δẋ_RL(3) 任务松弛, z(4) 零空间系数]` |
| 控制律 | — | `q̇ = J⁺(ẋ_nom + σ·Δẋ_RL) + B·z` |
| 门控 σ | [0,1] | smoothstep：安全→0，危险→1 |

### "弃车保帅"机制（四层防护）

| 层级 | 机制 | 位置 |
|------|------|------|
| 奖励层 | 动态跟踪权重：`w_track_eff` 随 d_obs 降低至 10% | `reward.py:_effective_track_weight` |
| 控制层 | σ 门控：危险时 RL 获得全部任务空间控制权 | `manipulator_env.py:step()` |
| 控制器层 | 斥力速度场：`v_rep = 0.5*(d_safe-d_obs)*direction` | `manipulator_env.py:_compute_task_velocity` |
| 惩罚层 | 碰撞惩罚：r_collision = -100（障碍物）-50（自碰） | `reward.py:compute` |

## 2. 训练策略：三阶段课程学习

### 阶段 1 — 纯跟踪热身（无障碍物）

**目标**：学会精确的末端轨迹跟随

| 参数 | 值 | 原因 |
|------|-----|------|
| 场景 | 无障碍（phase1_scenes.json） | 消除干扰，专注学习跟踪 |
| collision_term | False | 不提前终止 |
| lambda_dyn | 0.0 | 无力矩约束，降低探索难度 |
| steps | 100,000 | 简单任务快速收敛 |
| start_steps | 5,000 | 减少随机探索 |
| w_track | 3.0 | 全量跟踪权重 |
| success_bonus | 50.0 | 稀疏终点奖励 |

**收敛标准**：验证成功率 > 95%

### 阶段 2 — 完整避障训练

**目标**：在 5 个障碍物场景中同时完成跟踪与避障

| 参数 | 值 | 原因 |
|------|-----|------|
| 场景 | train_scenes.json（80 场景，5 障碍物） | 多样化障碍配置 |
| collision_term | True | 碰撞即终止 |
| lambda_dyn | 1.0 | 引入力矩正则化 |
| d_critical | 0.05m | 松弛阈值（障碍 < 5cm 时触发） |
| alpha_relax | 0.1 | 最小跟踪权重 = w_track × 10% |
| d_safe | 0.06m | 安全距离阈值 |
| w_obs | 5.0 | 障碍惩罚（占主导地位） |
| w_collision | 100.0 | 碰撞惩罚 |
| steps | 500,000 | 主训练阶段 |
| start_steps | 20,000 | 较长探索期适应新任务 |

**收敛标准**：成功率 > 80%，碰撞率 < 15%

### 阶段 3 — 鲁棒性微调

**目标**：增强泛化能力和力矩约束满足

| 参数 | 值 | 原因 |
|------|-----|------|
| lambda_dyn | 3.0 | 强化力矩约束 |
| alpha (SAC) | 0.05 | 降低探索噪声 |
| lr | 1e-4 | 小学习率精调 |
| steps | +200,000 | 从阶段2最佳模型继续 |

**收敛标准**：成功率稳定 > 85%，碰撞率 < 10%

## 5. "弃车保帅"机制验证

训练后需确认以下行为链：

```
末端接近障碍物 → d_obs ↓ → sigma ↑ → Δẋ_RL 幅值 ↑
→ 跟踪误差暂时 ↑（偏离轨迹）→ 末端绕过障碍物
→ d_obs 恢复 → sigma ↓ → 跟踪误差恢复正常 → 到达目标点
```

**验证脚本逻辑**：
- 选择已知有近距离障碍物的场景
- 记录每步的 d_obs、sigma、|Δẋ_RL|、tracking error
- 确认 sigma 与 d_obs 呈负相关
- 确认高 sigma 时的跟踪误差 > 低 sigma 时的跟踪误差（证明"牺牲"行为）

## 6. 收敛监控与调参指南

### 正常收敛信号

- Q-loss 持续下降
- Actor RL loss 震荡但趋势下降
- Physics loss 逐渐降低
- 回合奖励逐步提高
- d_obs 最小值 > 0

### 警告信号及对策

| 现象 | 诊断 | 对策 |
|------|------|------|
| 100K步后成功率为0% | 奖励信号不足以引导 | 增大 w_obs，检查奖励缩放 |
| Physics loss 比 actor loss 大 10× | 力矩约束过强 | 降低 lambda_dyn |
| 大部分回合碰撞终止 | 惩罚太弱或安全距离太近 | 增大 d_safe，减小碰撞惩罚 |
| Actor loss 出现 NaN | 梯度爆炸 | 降低 lr 或增大梯度裁剪 |
| 跟踪好但碰撞多 | 跟踪权重过高 | 增大 w_obs 或减小 w_track |
| 避障好但不到达目标 | 跟踪奖励不足 | 减小 w_obs 或增大 success_bonus |
